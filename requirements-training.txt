# lerobot requires numpy>=2.0, but ROS jazzy's cv_bridge.boost C-extension
# is built against numpy 1.x and crashes under 2.x. We side-step this by
# NOT installing ros-jazzy-cv-bridge — image decode happens via numpy
# directly in pai_teach. All other ROS C-extensions are ABI-tolerant.
#
# Torch / cuda: pick the wheel matching the training machine's CUDA driver.
# lerobot currently requires torch>=2.7,<2.12; let the resolver choose when
# installed alongside lerobot below.
# See https://pytorch.org/get-started/locally/
